## 1. Обзор технологий

Сайт построен на **Hugo** — генераторе статических сайтов. Это означает:

- Весь контент хранится в текстовых `.md` файлах (Markdown).
- Hugo компилирует их в HTML при каждом изменении.
- Готовый сайт — набор статических файлов без базы данных и серверной логики.
- Публикация происходит автоматически через **GitHub Actions** при пуше в ветку `main`.

**Минимальный набор инструментов для работы:**

| Инструмент | Назначение |
|-----------|-----------|
| Git | Управление версиями, отправка изменений |
| Hugo CLI | Локальный предпросмотр и сборка |
| Текстовый редактор | Редактирование `.md` и `.toml` файлов |

## 2. Локальная разработка

### Установка Hugo

```bash
# macOS
brew install hugo

# Linux (Debian/Ubuntu)
sudo apt install hugo

# Windows — скачать с https://github.com/gohugoio/hugo/releases
# Нужна расширенная версия: hugo_extended_*
```

Требуемая версия: **0.160.1** (указана в `.github/workflows/hugo.yml`).

### Запуск локального сервера

```bash
# Перейти в папку проекта
cd /путь/к/ngpu.org

# Запустить сервер с черновиками
hugo server -D

# Сайт откроется по адресу: http://localhost:1313
# Сервер автоматически перезагружает страницу при изменении файлов
```

### Сборка сайта вручную

```bash
hugo --minify
# Результат помещается в папку public/
```

## 3. Добавление новостей

Все новости хранятся в папке `content/news/`. Каждая новость — отдельный `.md` файл.

### Создание новости через Hugo CLI

```bash
hugo new news/nazva-novyny.md
```

### Создание вручную

Создайте файл `content/news/nazva-novyny.md` со следующей структурой:

```markdown
---
title: "Заголовок новости"
date: 2026-04-26T10:00:00+03:00
category: "Назва категорії"
author: "Ім'я Прізвище"
lede: "Короткий підзаголовок або анонс новини (1–2 речення)."
photo: "/img/news/nazva-foto.jpg"
photo_caption: "Підпис до фотографії"
photo_credit: "Фото · Ім'я автора або джерело"
tags: ["тег1", "тег2"]
featured: false
breaking: false
draft: false
---

Текст новини. Підтримується **Markdown**-розмітка.

## Підзаголовок

Ще один абзац тексту.
```

### Поля шапки (front matter)

| Поле | Обязательное | Описание |
|------|:-----------:|---------|
| `title` | да | Заголовок в браузере и в списке новостей |
| `date` | да | Дата и время публикации (влияет на сортировку) |
| `category` | нет | Категория отображается над заголовком |
| `author` | нет | Автор материала |
| `lede` | нет | Подзаголовок/анонс (отображается в карточке и под заголовком) |
| `photo` | нет | Путь к главному изображению |
| `photo_caption` | нет | Подпись к фото |
| `photo_credit` | нет | Кредит / источник фото |
| `tags` | нет | Список тегов |
| `featured` | нет | `true` — показывать в блоке «Главное» на главной |
| `breaking` | нет | `true` — помечает новость как срочную |
| `draft` | нет | `true` — черновик, не публикуется |

### Доступные шорткоды внутри текста

```markdown
{{< kicker >}}Рубрика чи тема{{< /kicker >}}

{{< pullquote author="Ім'я Прізвище, посада" >}}
Цитата з матеріалу.
{{< /pullquote >}}

{{< photo src="/img/news/foto.jpg" caption="Підпис" credit="Фото · Джерело" >}}

{{< stats >}}

{{< numbered >}}
1. **Пункт перший** — опис
2. **Пункт другий** — опис
{{< /numbered >}}
```

## 6. Обновление настроек сайта

Главный файл конфигурации: **`hugo.toml`** в корне проекта.

### Основные параметры

```toml
[params]
  description = "Опис сайту для мета-тегів"
  established  = 1992          # Рік заснування
  members      = "174 312"     # Кількість членів профспілки
  primaries    = 412           # Кількість первинних організацій
  regions      = 22            # Кількість регіональних організацій
  hotline      = "+380503208035"

  accent = "blue"    # Колір акценту: ochre | blue | red | forest | ink
  mode   = "light"   # Тема: light | dark
```

### Контактная информация

```toml
[params.contacts]
  address = "Київ, Майдан Незалежності, 2, к. 617"
  phone1  = "+38 (044) 205-74-63"
  phone2  = "+38 (044) 205-74-60"
  email   = "info@ngpu.org.ua"
```

### Цветовые акценты

| Значение | Цвет |
|---------|------|
| `ochre` | Охра (золотисто-жёлтый) |
| `blue` | Синий (по умолчанию) |
| `red` | Красный |
| `forest` | Тёмно-зелёный |
| `ink` | Чёрный/чернильный |

## 7. Добавление изображений

### Где хранить изображения

| Папка | Назначение |
|-------|-----------|
| `static/img/` | Изображения, используемые в контенте (фото для новостей) |
| `static/img/news/` | Фото для новостей (рекомендуется) |
| `content/img/` | Изображения, привязанные к конкретным материалам |

### Как добавить фото

1. Скопируйте файл в `static/img/news/nazva-foto.jpg`
2. В шапке новости укажите путь: `photo: "/img/news/nazva-foto.jpg"`

### Рекомендации по размеру

- Ширина: **1200 px** или более
- Формат: **JPEG** (для фото), **PNG** (для логотипов/графики), **WebP** (для оптимизации)
- Размер файла: до **300 KB** для быстрой загрузки

## 8. Навигационное меню

Пункты меню настраиваются в `hugo.toml`:

```toml
[menu]
  [[menu.main]]
    identifier = "home"
    name       = "Головна"
    url        = "/"
    weight     = 1

  [[menu.main]]
    identifier = "news"
    name       = "Новини"
    url        = "/news/"
    weight     = 2
```

- **`name`** — текст пункта меню
- **`url`** — ссылка
- **`weight`** — порядок (меньше = левее)

Чтобы добавить новый раздел:
1. Создайте папку в `content/` (например `content/members/`)
2. Создайте файл `content/members/_index.md`
3. Добавьте пункт меню в `hugo.toml`

## 10. Публикация (деплой)

### Автоматическая публикация

Любые изменения, отправленные в ветку **`main`**, автоматически собираются и публикуются через GitHub Actions.

```bash
# Стандартный рабочий процесс

# 1. Убедитесь, что находитесь на нужной ветке
git checkout main

# 2. Добавьте изменённые файлы
git add content/news/nova-novyna.md static/img/news/foto.jpg

# 3. Создайте коміт
git commit -m "Додано новину про ..."

# 4. Відправте зміни
git push origin main
```

После пуша GitHub Actions автоматически:
1. Установит Hugo v0.160.1 и Dart Sass
2. Запустит `hugo --minify`
3. Опублікує папку `public/` на GitHub Pages

Проверить статус деплоя можно во вкладке **Actions** репозитория на GitHub.

### Рабочий процесс с черновиками

Чтобы работать над материалом, не публикуя его:

1. Установите `draft: true` в шапке файла
2. Локально проверяйте с флагом `hugo server -D`
3. Когда готово — измените на `draft: false` и сделайте пуш

## 12. Часто задаваемые вопросы

**Как проверить сайт перед публикацией?**
Запустите `hugo server -D` и откройте `http://localhost:1313`. Изменения отображаются мгновенно.

**Новость опубликована, но не отображается на сайте?**
- Проверьте, что `draft: false` (или поле отсутствует).
- Убедитесь, что дата (`date`) не в будущем. Hugo не показывает материалы с будущей датой.
- Проверьте статус деплоя в разделе Actions на GitHub.

**Как удалить новость?**
Удалите соответствующий `.md` файл и сделайте коміт + пуш.

**Как изменить порядок новостей?**
Сортировка автоматическая — по полю `date` (новые сверху). Чтобы закрепить материал — используйте `featured: true`.

**Как добавить новый раздел в меню?**
Создайте папку в `content/`, добавьте `_index.md` и добавьте пункт `[[menu.main]]` в `hugo.toml`.

**Где посмотреть логи деплоя?**
GitHub → репозиторий → вкладка **Actions** → последний workflow.

**Сайт «сломался» после изменений — что делать?**
```bash
# Откатиться к предыдущему коміту
git revert HEAD
git push origin main
```
